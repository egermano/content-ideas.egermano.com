### Experiment 1 â€” Refactor a Legacy Code Snippet
- **Objective:** To demonstrate the effectiveness of an AI-powered refactoring tool in a real-world scenario by improving a piece of legacy code.
- **Stack:** A small, self-contained legacy project (e.g., an old jQuery-based JavaScript file or a complex Python script), and an AI refactoring tool like Sourcery.
- **Steps:**
  1. Identify a suitable piece of legacy code that is functional but poorly written (e.g., long functions, nested loops, unclear variable names).
  2. Establish a baseline by running a code quality analysis tool (e.g., SonarLint) to get metrics on code smells, complexity, and maintainability.
  3. Use an AI refactoring tool to automatically suggest and apply improvements to the code.
  4. Re-run the code quality analysis to measure the improvements in the same metrics.
  5. Ensure the refactored code still functions as intended by running any existing tests or performing manual UAT.
- **Expected Outputs:** A clear before-and-after comparison of the code, with quantifiable improvements in code quality metrics.
